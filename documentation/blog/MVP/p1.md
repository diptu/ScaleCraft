# Building a Modern URL Shortener with NestJS, Prisma, and Supabase
 In this tutorial, we will architect a robust, scalable URL shortener using the modern "Gold Stack": NestJS for the backend, Prisma as our ORM, and Supabase (PostgreSQL) for our data layer.

## 0. Prerequisites {#prerequisites}

Before we begin, ensure you have the following installed:
* **Node.js** (v18 or higher)
* **npm** or **pnpm**
* A basic understanding of **TypeScript** and **Dependency Injection**.

---
## 1.Project Scaffolding {#project-scaffolding}
e start by using the **Nest CLI**. This is the industry standard for creating enterprise-grade Node.js applications because it enforces a modular architecture from day one.

### 1.1 Install Nest CLI
If you haven't installed it globally, run:
```bash
npm i -g @nestjs/cli
```

### 1.2 Initialize the Application
Generate the project structure. We'll name our project url_shortener.
```bash
nest new url_shortner
```
### 1.3 Select Package Manager
During the installation, you will be prompted to choose a package manager. For this project, we recommend npm.

Once the installation is complete, navigate into your project:

```bash
cd url_shortner
```
### 1.4 Project Structure Overview

At this stage, your src/ directory contains:

- main.ts: The entry point of the application.

- app.module.ts: The root module of the application.

- app.controller.ts: A basic controller with a single route.

- app.service.ts: A basic service with a single method.

## 3 Versioning & Module Routing {#versioning--module-routing}

### 3.1 Create Root V1Module
```bash
nest generate module modules/v1
```
### 3.2 Your src folder should now look like:
```bash
src/
 ├── modules/
 │    └── v1/
 │         └── v1.module.ts
 ├── app.module.ts
 └── main.ts


```
### 3.3 Update the Root Module

```ts
import { Module } from '@nestjs/common';

@Module({
  imports: [],
  controllers: [],
  providers: [],
})
export class AppModule {}
```

### 3.4 Register V1Module in AppModule

```ts
import { Module } from '@nestjs/common';
import { V1Module } from './modules/v1/v1.module';


@Module({
  imports: [V1Module],
  controllers: [],
  providers: [],
})
export class AppModule { }

```
### 3.5 Add Global API Prefix /v1 to your src/main.ts

```ts
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  app.setGlobalPrefix('v1');

  await app.listen(3000);
}
bootstrap();

```
### 3.6 Restart Server
```bash
npm run start:dev

```
now visit :`http://localhost:3000/v1`

You should see:`Cannot GET /v1`

## 4 Create the URLs Module

### 4.1 Generate URLs Module (CLI)

```bash
nest g module modules/v1/urls
```
Now, you should have

```bash
src/modules/v1/urls/
 └── urls.module.ts
```

### 4.2 Generate URLs Controller

Now generate the controller without routes yet:

```bash
nest g controller modules/v1/urls
```
This creates: `src/modules/v1/urls/urls.controller.ts`

Now Visit: `http://localhost:3000/v1/urls`

You should see: `Cannot GET /v1/urls`
